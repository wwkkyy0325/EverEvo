# 工作流

工作流引擎让你用可视化的 DAG 编排模型管线：把多个节点连起来，串起「预处理 → 推理 → 后处理」或更复杂的流程，支持条件分支与模板变量。

## 节点类型

- **输入节点**：定义工作流的入参，下游通过模板引用。
- **LLM 节点**：调用 LLM 生成文本，输出归一化为 `{output, content, raw}`。
- **Agent 节点**：委派给一个本地 Agent 人格执行。
- **工具节点**：调用任意已注册工具。
- **条件节点**：按表达式分支。
- **输出节点**：汇聚最终结果。

## 模板变量

节点配置里的字符串字段支持双花括号模板，路径首段可以是**节点 ID、节点标题、或顶层输入键**三者之一：

- `{{n1.field}}` —— 引用某节点的输出字段
- `{{节点标题.field}}` —— 用中文标题也行
- `{{fieldName}}` —— 引用顶层输入

例如 LLM 节点的 prompt 写 `请总结：{{inputNode.text}}`，运行时会用上游真实输出替换。

## 执行

- 点「执行」运行，实时看到每个节点的状态与进度（LLM / Agent 长节点会有「正在生成…」心跳，不会静默卡住）。
- 用 `workflow_status` 轮询执行进度。
- 后端任何变更会经事件实时推给前端，编辑器即时刷新。

## 适用场景

- 批量文本处理：输入 → LLM 摘要 → 输出。
- 多模型协作：句向量召回 → LLM 基于召回内容回答（RAG 式管线）。
- Agent 编排：主 Agent 调度多个专用 Agent 协同完成任务。
